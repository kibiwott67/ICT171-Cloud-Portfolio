## Project Overview

MedCare is a multi-page healthcare information website deployed on a Linux virtual machine in Microsoft Azure. It provides health tips, a BMI calculator, a PHP-based contact form, and a live server status page. The server is configured from scratch using Infrastructure as a Service (IaaS) — no pre-configured images or bundles were used.

**Features implemented:**
- 4-page static HTML/CSS website with responsive design
- PHP contact form with server-side validation and logging
- Bash monitoring script that generates both a terminal dashboard and a public HTML status page
- SSL/TLS certificate via Let's Encrypt
- GoDaddy DNS A records pointing to Azure static IP

## 1. Azure VM Setup

Log in to [portal.azure.com](https://portal.azure.com) and navigate to **Virtual Machines > Create > Azure Virtual Machine**.

**Settings used:**

| Field | Value |
|-------|-------|
| Resource Group | ICT171 |
| VM Name | medcare |
| Region | East Asia |
| Image | Ubuntu Server 22.04 LTS — x64 Gen2 |
| Size | Standard_B1s (1 vCPU, 1 GiB RAM) |
| Authentication | SSH public key |
| Inbound ports | SSH (22) |

After the VM is created:

1. **Download the `.pem` private key** when prompted — this cannot be retrieved later.
2. Go to **Networking > Inbound port rules** and add:
   - Port **80** (HTTP) — Priority 310
   - Port **443** (HTTPS) — Priority 320

### Make the IP address static

In Azure Portal: **VM > Networking > Public IP address (click the link) > Configuration > Assignment: Static > Save**

Note the static IP

## 4. Apache2 + PHP Installation

Install Apache2 and PHP:

```bash
sudo apt install apache2 php libapache2-mod-php -y
```

Verify Apache is running:

```bash
sudo systemctl status apache2
```

Expected output includes `active (running)` in green.

Enable Apache to start automatically on reboot:

```bash
sudo systemctl enable apache2
```

Test by opening a browser and navigating to `http://YOUR_VM_IP`. The Apache2 default page should appear.

Verify PHP is working:

```bash
php -v
```

## 5. Website Deployment

Remove the Apache default page and deploy the MedCare website:

```bash
sudo rm /var/www/html/index.html
```

Copy all website files to the web root. The MedCare site consists of:

```
/var/www/html/
├── index.html          # Home page
├── services.html       # Services page with BMI calculator
├── health-tips.html    # Filterable health tips
├── contact.html        # Contact page
├── submit.php          # PHP form handler
├── style.css           # Stylesheet
├── main.js             # JavaScript (tips, BMI, filter)
└── status.html         # Generated by monitoring script
```

Upload files via SCP from your local machine:

```bash
scp -i ~/Downloads/medcare_key.pem -r ./medcare-site/* azureuser@YOUR_VM_IP:/var/www/html/
```

Set correct ownership:

```bash
sudo chown -R www-data:www-data /var/www/html/
sudo chmod -R 755 /var/www/html/
```

Verify the site loads at `http://YOUR_VM_IP`.

## 7. SSL/TLS with Let's Encrypt

> **Prerequisite:** DNS must be propagated before running Certbot.

Install Certbot and the Apache plugin:

```bash
sudo apt install certbot python3-certbot-apache -y
```

Obtain and install the certificate:

```bash
sudo certbot --apache -d allankibiwott.net -d www.allankibiwott.net
```

When prompted:
- Enter an email address for renewal alerts
- Agree to the Terms of Service
- Choose **option 2 (Redirect)** to force HTTPS on all traffic

Verify HTTPS is active:

```bash
sudo systemctl status certbot.timer
```

Open `https://allankibiwott.net` in a browser — the padlock icon confirms the certificate is valid.

Certificates renew automatically every 90 days. Test the renewal process:

```bash
sudo certbot renew --dry-run
```

## 8. PHP Contact Form Setup

## 9. Monitoring Script

`medcare_monitor.sh` is the scripting component of this assignment. It is an original script that checks server health and generates both a terminal dashboard and a public HTML status page.

### What the script does

1. Checks Apache2 service status
2. Makes HTTP and HTTPS requests to `allankibiwott.net` and records the response codes
3. Verifies PHP is available
4. Reads CPU usage, RAM usage, and disk usage
5. Draws ASCII bar charts in the terminal (colour-coded: green/yellow/red)
6. Appends a timestamped log entry to `/var/log/medcare_monitor.log`
7. **Generates `/var/www/html/status.html`** — a public, auto-refreshing HTML dashboard verifiable at `https://allankibiwott.net/status.html`

### Setup

```bash
cp medcare_monitor.sh /home/azureuser/medcare_monitor.sh
chmod +x /home/azureuser/medcare_monitor.sh
```

Create the log file:

```bash
sudo touch /var/log/medcare_monitor.log
sudo chmod 666 /var/log/medcare_monitor.log
```

Run the script:

```bash
sudo /home/azureuser/medcare_monitor.sh
```
